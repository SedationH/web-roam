## 设计原则

开放-封闭原则

软件实体（类、模块、函数）等应该是可以扩展的，但是不可修改。

在面向对象中，若违反其他任何原则，大多数同时违反开放-封闭原则



单一职责原则

一个方法、对象制作一件事情。

职责被定义为“引起变化的原因”，单一职责为了保持设计的低耦合和高内聚



最少知识原则

软件实体应当尽可能少的与其他实体发生相互作用。
