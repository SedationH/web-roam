> 我以前在学编译原理的时候就感觉非常懵逼，像是在上数学课，一上来引入了一大堆定义，各种推导，结果到做实验的时候写词法分析器，连状态转换图都没画就写完了，至于NFA->DFA,DFA的最小化更是一点都没用到。
>
> 后面才发现编译原理的那一大堆概念是为了自动生成词法分析器，比方说lex/flex，写正则就可以自动生成一个词法分析器了，实际上如果你自己手写词法分析器是用不到那些概念的。
>
> 学校那么教我估计是因为那些概念好出题....
>
> 我觉得完全没有基础的话可以试试写一个markdown解析器，然后生成对应的HTML，毕竟markdown语法非常简单。
>
> \# 这是一级标题
>
> \## 这是二级标题
>
> **这是加粗的文字**
>
> \> 这是引用的内容
>
> 即便没有学过编译原理也可以看出，如果句子的开头是一个#，那就继续往前读，如果读到的下一个字符不是#，那就停下来数一数刚刚读到的#有多少个，有一个#那就生成一个<h1></h1>，两个#就是h2标签，以此类推。然后读取后面的字符直到读到换行符为止，把剩余部分的内容填充到<h1></h1>里。然后[# 这是一级标题]就变成了[<h1>这是一级标题</h1>]，在写markdown解析器的时候肯定会遇到一些问题，然后再去看编译原理的概念，就会发现“wow，原来我写的那个if else就是有限状态自动机”。
>
> 我非常看重理论在实际中的用途，如果一个理论我在实际中用不到，那么学习起来就会痛不欲生。为了好好学习我给编译原理找了一个用处，用markdown来做PPT。众所周知做PPT的时候排版是一个很耗时的环节，我希望可以设计一个模板然后直接敲markdown就可以自动生成PPT了，无需排版（当然生成PPT是不可以的，准确地说是生成一个长得很像PPT的HTML文件，可以部署到Github Pages一类服务器然后用链接就可以访问）。
>
> markdown语法可以和下面课件的很多地方相对应，比方说标题"LL(1)分析算法"可以用#转换成h1标签来表示，蓝色字体部分可以用**xx ******来表示，无序列表可以用- xxx来表示。
>
> ![img](https://pic4.zhimg.com/50/v2-2dcf64f2d07842a61a766447554f3127_hd.jpg?source=1940ef5c)![img](https://pic4.zhimg.com/80/v2-2dcf64f2d07842a61a766447554f3127_720w.jpg?source=1940ef5c)
>
> 但是很明显markdown的基本语法有些简单，有些效果实现不了，比方说我想让PPT里的某个元素可以被点击，然后弹出一个对话框展示一些内容。我希望在markdown里写一句
>
> \##这句话可以被点击@click(alert("哈哈哈"))
>
> 就可以实现上面的需求，这样就需要扩展markdown语法了，然后生成的HTML是：
>
> <h2 onclick="alert('哈哈哈')">这句话可以被点击</h2>
>
> 这样就是编译原理的一个很简单的应用，当然这件事我没有继续做下去，因为后面发现了nodePPT...
>
