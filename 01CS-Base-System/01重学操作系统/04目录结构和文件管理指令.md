## shell

> In [computing](https://en.wikipedia.org/wiki/Computing), a **shell** is a [user interface](https://en.wikipedia.org/wiki/User_interface) for access to an [operating system](https://en.wikipedia.org/wiki/Operating_system)'s services. In general, operating system shells use either a [command-line interface](https://en.wikipedia.org/wiki/Command-line_interface) (CLI) or [graphical user interface](https://en.wikipedia.org/wiki/Graphical_user_interface) (GUI), depending on a computer's role and particular operation. It is named a shell because it is the outermost layer around the operating system.[[1\]](https://en.wikipedia.org/wiki/Shell_(computing)#cite_note-Economist-1) [[2\]](

简单来说，是一个命令行界面，用户可以通过这个命令行界面来操作系统，执行指令



## Linux对文件 目录的抽象

常见文件类型： 通过`ls -F`可以看到他们的文件类型,以 -> 所示的符号结尾

1. 普通文件 如一个文本文件 -> 无
2. 目录文件 如 /  -> /
3. 可执行文件 如 rm 指令 -> *
4. 管道文件 -> |
5. Socket文件 -> =
6. 软链接文件 -> @
7. 硬链接文件



Socket是网络插座，是用户端和服务器端之间同步数据的接口。

Linux把所有的设备都抽象成了文件。因为设备需要不断和操作系统进行IO，文件也是如此，这让整体系统设计变得高度统一。
